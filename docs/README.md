## 1.1　プロジェクト概要

| 項目 | 内容 |
|---|---|
| サービス名 | 南高祭 公式サイト |
| 開催日 | 2025年9月13日（土）・14日（日） |

### 現状の課題

- 当日配布の紙のパンフレットは更新できず、変更情報が伝わらない

| 情報の鮮度 | リアルタイムで更新される展示情報・お知らせ通知 |
| 混雑状況 | 待ち時間をリアルタイム表示（マップと連携） |
| シフト管理 | 希望収集→自動割り当て→共有までをシステム化 |
| 道案内 | インタラクティブな校舎マップでフロア別に案内 |

# 第3章　セキュリティ設計

> ⚠ **本章は許可申請の審査において特に重要な箇所です。**

## 3.1　認証システム

### 仕組み

本サービスは **Supabase Auth**（業界標準の認証基盤）を採用しています。

```
ログインフロー:
  1. ユーザーがメール・パスワードを入力
  2. Supabase が認証を行い JWT（JSON Web Token）を発行
  3. JWT はブラウザの Cookie に保存（HttpOnly・Secure フラグ）
  4. 以後のすべてのリクエストに JWT が自動付与される
  5. サーバーサイドで JWT を検証してから処理を実行
```

### パスワードの保護

- パスワードは **bcrypt** でハッシュ化して保存（平文では保存しない）
- パスワードは最低8文字を必須とし、強度インジケーターで誘導
- パスワードリセットはメールリンク経由のみ

### セッション管理

- JWT の有効期限は Supabase のデフォルト設定に準拠（1時間で自動更新）
- ログアウト時は JWT を無効化しセッションを完全破棄

## 3.2　なりすまし防止

### ロールベースアクセス制御（RBAC）

ユーザーのロール（`admin` / `editor` / `student`）は **サーバーサイドのデータベース** に保存されています。

```
なりすましが不可能な理由:
  ✗ ロール情報をブラウザの Cookie や localStorage に保存しない
  ✓ API アクセス時に毎回サーバーでロールを照合する
  ✓ JWT に含まれる user_id でデータベースのロールを確認

例: editor が admin のページにアクセスしようとした場合
  → サーバーが JWT から user_id を取得
  → profiles テーブルで role = 'editor' を確認
  → 403 Forbidden を返却（アクセス不可）
```

### 担当クラスの制限

editor は `exhibit_editors` テーブルで許可された展示 ID のみ編集可能です。他クラスの情報を改ざんすることは技術的に不可能です。

```
例: 高1-1 の editor が 高2-3 の情報を書き換えようとした場合
  → API が JWT の user_id を確認
  → exhibit_editors テーブルで「高2-3 の exhibit_id」への許可がないと判定
  → 403 Forbidden を返却
```

### 行レベルセキュリティ（RLS）

PostgreSQL の **RLS（Row Level Security）** により、データベース層でもアクセス制御を実施しています。

```sql
-- 例: シフト表は自分のクラスのメンバーのみ参照可能
CREATE POLICY "self read on student_exhibits"
  ON student_exhibits FOR SELECT TO authenticated
  USING (user_id = auth.uid());
```

これにより、仮にアプリケーション層のバグがあってもデータベース層で不正アクセスをブロックします。

## 3.3　スタンプラリーの不正防止

### QR コードの仕組み

スタンプ用 QR コードは **HMAC-SHA256** による署名付き URL を使用しています。

```
QR URL の構造:
  /stamp?e={展示ID}&w={時間ウィンドウ}&h={HMAC署名(16文字)}

時間ウィンドウ = Unix時刻 ÷ 60秒 の整数（1分ごとに変化）

検証フロー:
  1. QR を読み取り、サーバーに送信
  2. サーバーが展示の stamp_secret（DB に保存）で HMAC を再計算
  3. QR の h パラメータと一致すれば正規 QR と判定
  4. ±1 ウィンドウ（最大2分）を許容してスキャン時の時計ズレに対応
```

**不正が不可能な理由:**
- `stamp_secret` はサーバーのみが知る秘密鍵（クライアントには非公開）
- 秘密鍵なしに正規の `h` を生成することは計算上不可能
- 1分ごとに QR が変化するため、古い QR の写真は最大2分後に無効化

### スタンプの重複防止

- スタンプは `(user_id, exhibit_id)` の組み合わせで UNIQUE 制約
- 同じ展示を複数回スキャンしても1回しか記録されない

## 3.4　プッシュ通知のセキュリティ

- FCM トークンはユーザーの `user_id` に紐付けて管理
- 通知の送信は **service_role_key**（サーバー専用秘密鍵）を持つ API のみが実行可能
- ユーザーは自分の通知設定のみ変更可能（他人の通知を操作不可）

## 3.5　サービスロールキーの保護

API の一部は Supabase の `service_role_key`（RLS をバイパスできる管理者権限キー）を使用しますが：

- このキーは **環境変数**（`.env.local`）にのみ保存
- フロントエンドのコードには一切含まれない
- サーバーサイドの API Route からのみアクセス可能
- Vercel のシークレット管理機能で本番環境に設定

## 3.6　画像・著作権保護

ユーザーがページをスクリーンショット保存・コンテキストメニューから保存することを防ぐ対策を実装：

- すべての `<img>` タグに `pointer-events: none` + `-webkit-user-drag: none` を設定
- 右クリックメニューを JavaScript でブロック
- テキスト選択禁止（入力欄は除く）

# 第4章　来場者向けガイド

## 4.1　アクセス方法

1. スマートフォンのブラウザ（Safari / Chrome）でサイトにアクセス
2. ホーム画面への追加（推奨）が表示された場合は追加するとアプリとして利用可能

> **💡 スタンプラリーと通知機能を利用するには、ホーム画面への追加（PWA インストール）が必要です**

### iOS（iPhone）でのインストール方法

```
1. Safari でサイトを開く
2. 画面下部の「共有」ボタン（□に↑のマーク）をタップ
3. 「ホーム画面に追加」を選択
4. 右上の「追加」をタップ
```

### Android での インストール方法

```
1. Chrome でサイトを開く
2. 右上メニュー（⋮）→「ホーム画面に追加」を選択
3. 表示されたダイアログで「追加」をタップ
```

## 4.2　校舎マップ

*[スクリーンショット: マップページ全体表示]*

### 基本操作

| 操作 | 内容 |
|---|---|
| ピンチイン/アウト | 拡大・縮小 |
| ドラッグ | 移動 |
| 教室をタップ | 展示情報のボトムシートを表示 |

*[スクリーンショット: 展示詳細ページ（画像・説明・メニュー）]*

展示をタップすると詳細ページへ遷移します。

- 動画・画像ギャラリー
- フードの場合：メニュー一覧・価格・在庫状況
- いいねボタン
- 「予定に追加」ボタン
- コメント欄（承認制）

## 4.4　お知らせ

### 一覧ページ

- 各クラスから投稿された最新情報が新着順に表示
- 未読のお知らせにはオレンジのバッジ
- 「重要」マークつきのお知らせは上部に強調表示

### 詳細ページ

- 本文・リンク・画像・動画を含む詳細表示
- 開いた時点で既読マーク

### プッシュ通知

1. 「通知」タブ → 「南高祭 公式アナウンス」を ON
2. 特定の展示の通知を ON にすると、その展示のお知らせ投稿時に通知が届く

## 4.5　スタンプラリー

> **PWA インストール（ホーム画面に追加）が必要です**

*[スクリーンショット: スタンプラリーページ（グリッド表示）]*

### 参加方法

1. ホーム画面のアイコンからアプリを起動
2. 「スタンプ」タブをタップ
3. 「QR をスキャン」ボタンをタップ
4. 展示のQRコードを読み取る
5. スタンプが押される（グリッドに展示の写真が表示される）

### スタンプの特典

- 集めた展示に対して**人気投票**が可能

QR コードは**1分ごとに変化する使い捨てコード**です。他の来場者の QR 画像をコピーしても2分以内に無効化されます（詳細は第3章参照）。

## 4.6　人気投票

スタンプを集めた展示に1票を投票できます。

- **条件**: スタンプラリーで1つ以上の展示を訪問
- **操作**: 投票したい展示を選んで「投票する」ボタン
- **変更**: 投票後も投票先を変更できます
- **結果公開**: 管理者が設定したタイミングで公開

## 4.7　予定管理

*[スクリーンショット: 予定タイムラインページ]*

当日のスケジュールを 8:30〜16:30 のタイムラインで管理できます。

### 予定の追加方法

**方法①: 各ページから追加**
- 展示詳細・催し物・軽音のページの「📅 予定に追加」ボタンをタップ
- 通知タイミング（5分前/10分前/15分前/30分前/1時間前）を選択

**方法②: 予定ページから追加**
- 「予定」タブ右上の「＋」ボタン
- タイトル・日付・開始/終了時刻・場所・通知を設定

### 同時間帯の表示

同じ時間帯に複数の予定がある場合は**横に並べて**表示されます。

### 通知

- 設定した時間前にブラウザ通知で知らせる
- 通知には展示名・場所が表示される

> **注意**: 通知はアプリを起動している間のみ動作します

## 4.8　フード情報

「フード」リンクから模擬店・食堂の情報を確認できます。

- **人気ランキング**: 販売数上位のメニューを表示
- **メニュー一覧**: クラス別に価格・在庫・販売状況を表示
- 「販売中のみ」フィルターで在庫あり商品のみ表示可能

## 4.9　軽音・催し物

**軽音楽部（バンドステージ）**
- 出演バンド一覧・メンバー・スケジュール
- 現在演奏中のバンドには「LIVE」バッジ

**催し物**
- ダンス・演劇・合唱など特別イベントのスケジュール
- 「予定に追加」ボタンで登録可能

## 4.10　通知設定

「通知」タブから各種通知設定ができます。

| 通知の種類 | 設定場所 |
|---|---|

# 第5章　生徒（student）向けガイド

## 5.1　アカウント作成

1. 管理画面（`/admin`）→「アカウント作成」
2. メールアドレス・パスワード（8文字以上）を入力
3. 「アカウントを作成」ボタン
4. 初回ログイン時にプロフィール（名前・学年・クラス・出席番号）を設定

> アカウント作成後のロールは `student` です。担当の先生（editor または admin）にクラスへの割り当てを依頼してください。

## 5.2　シフトアンケート

*[スクリーンショット: シフトアンケートページ]*

担当クラスのシフトコマが設定されたら、希望時間帯を回答します。

### 回答方法

1. 管理画面 → サイドバー「シフト管理」→「シフトアンケート」
2. 土曜日・日曜日それぞれのコマに対して3択で回答：
   - **◎ 入りたい**: この時間帯に入りたい
   - **△ どちらでもいい**: どちらでも対応可能
   - **✕ 入れない**: この時間帯には入れない（絶対に割り当てられない）
3. 「回答を保存する」ボタン

> **重要**: ✕ を選択したコマには絶対にシフトが割り当てられません。確実に都合が悪い場合のみ使用してください。

### 回答の変更

締め切りなし。「シフトアンケート」ページから何度でも変更できます。

## 5.3　シフト表の確認

1. 管理画面 → 「シフト表」
2. 土/日ボタンで日付を切り替え
3. 自分のコマはオレンジ色のアバターで表示

### シフト通知の設定

1. シフト表ページ右上の 🔔 ボタンをタップ
2. 「何分前に通知するか」を選択（5/10/15/30/60分前、またはなし）
3. 「設定する」ボタン
4. 以降は「予定」タブを開くと通知がスケジュールされる

# 第6章　担当者（editor）向けガイド

## 6.1　ログイン

管理画面（`/admin`）にアクセス → メールアドレス・パスワードを入力

> **editor は admin または別の editor に「担当展示の割り当て」をしてもらう必要があります。割り当てがない場合は展示情報を編集できません。**

## 6.2　展示情報の編集

### 一覧ページ（`/admin/edit`）

担当展示の一覧から対象をクリック。

### 詳細編集ページ（`/admin/edit/{id}`）

タブ構成：

**① 基本情報タブ**
- 展示名・キャッチコピー・説明文
- 表紙画像・サムネイル
- 場所・フロア

**② コンテンツタブ**
- セクション（見出し＋本文＋画像/動画）の追加・並び替え
- フードの場合：メニュー管理（名前・価格・説明・在庫・販売中フラグ・販売数）

**③ 設定・クイックタブ**
- 待ち時間の更新（待ち組数 × 1組あたりの時間）
- スタンプラリー参加設定
- いいね数表示設定

### 「保存する」ボタン

ページ上部・下部に配置。変更を確定します。

## 6.3　お知らせの管理

### お知らせ一覧（`/admin/notices`）

投稿済みのお知らせ一覧。新規作成ボタン。

### お知らせ作成・編集（`/admin/notices/{id}`）

| 項目 | 説明 |
|---|---|
| タイトル | お知らせのタイトル（最初の1行から自動設定） |
| 本文 | テキスト・リンク・改行を組み合わせた内容 |
| 画像・動画 | メディアの添付（複数可） |
| 重要マーク | ON にすると赤く強調表示される |

「投稿する」ボタンを押すと、購読しているユーザーにプッシュ通知が送信されます。

## 6.4　かんたん更新（Quick ページ）

展示管理画面の「かんたん更新」ボタン（または `/admin/quick/{id}`）で当日の素早い更新が可能。

- 待ち時間の更新
- スタンプ QR の表示
- フードメニューの在庫・販売数・販売状況の更新
- コメントの承認・削除

## 6.5　メンバー管理（シフト用）

1. 管理画面 → 「シフト管理」→「メンバー管理」
2. student アカウントを持つ生徒をクラスに割り当てる

**割り当て方法:**
- 学年・クラスでフィルタリング（★マークは担当クラスと一致するフィルター）
- 「表示中を一括追加」ボタンでまとめて割り当て
- 個別にタップして ON/OFF

## 6.6　シフト設定

1. 管理画面 → 「シフト編集」
2. タブ①「時間設定」でコマを生成

| 設定項目 | 説明 |
|---|---|
| 開始時刻 / 終了時刻 | シフトの範囲（例: 9:00 〜 17:00） |
| コマの長さ | 1コマ何分か（例: 30分） |
| 必要人数（一括） | 全コマ共通の必要人数 |

「コマを生成する」ボタンで自動生成。個別コマの必要人数も後から変更可能。

## 6.7　シフト割り当て

### アンケート確認タブ

*[スクリーンショット: アンケート結果マトリクス（◎△✕）]*

生徒全員の希望をマトリクスで確認できます。

| 記号 | 意味 |
|---|---|
| ◎ | 入りたい |
| △ | どちらでもいい |
| ✕ | 入れない |

### シフト割り当てタブ

**自動割り当て（推奨）**

「🤖 自動割当」ボタンを押すと以下のアルゴリズムで自動割り当てを実行：

```
割り当てアルゴリズム（2段階グリーディ法）:
  1. 充足率（◎人数÷必要人数）が低いコマを優先
  
  第1段階: ◎（入りたい）の人を上限人数内で割り当て
  第2段階: △（どちらでもいい）の人で補填
  必要に応じて上限を緩和して再試行
  
  ✕（入れない）の人は絶対に割り当てない
```

実行後、結果メッセージで以下を確認できます：
- 割り当て件数
- 人数不足のコマ（警告）
- 基準より多く割り当てられた人（📌）

**手動割り当て**

各コマの「＋」ボタンから生徒を選択。アンケート回答（◎△✕）と既存割り当てコマ数を参考に選択できます。

「💾 割当を保存」で確定。

# 第8章　個人情報・プライバシーポリシー

## 8.1　収集する情報

| 情報 | 対象 | 利用目的 |
|---|---|---|
| メールアドレス | ログインユーザー | 認証・パスワードリセット |
| 名前・学年・クラス・出席番号 | ログインユーザー | シフト管理・表示名 |
| スタンプ取得記録 | 来場者（匿名ID） | スタンプラリー・投票機能 |
| 個人予定データ | 来場者（匿名ID） | 予定管理機能 |
| いいね・投票記録 | 来場者（匿名ID） | 機能提供 |
| FCM トークン | 通知同意ユーザー | プッシュ通知送信 |

## 8.2　情報の管理

- すべてのデータは **Supabase**（データセンター: 日本またはアジア太平洋リージョン）に保管
- パスワードは bcrypt でハッシュ化して保存（平文では保管しない）
- ログインユーザーの情報は当事者以外の利用者からは閲覧不可

## 8.3　来場者の匿名性

ログイン不要のユーザーは、端末のローカルストレージに保存された **ランダムな UUID**（例: `a1b2c3d4-...`）のみで識別されます。

- 氏名・メールアドレス等の個人を特定できる情報は収集しない
- スタンプ・予定・投票記録はこの UUID に紐付き、**本人しか閲覧できない**

## 8.4　データの保存期間・削除

- 文化祭終了後、個人データの削除に関しては学校の情報管理規定に従う
- ユーザーはアカウント削除をリクエストすることができる（admin に依頼）

## 8.5　第三者提供

収集した個人情報は以下の場合を除き、第三者に提供しない：

- 法令に基づく場合
- 人の生命・身体または財産の保護のために必要な場合

## 8.6　使用する外部サービス

| サービス | 用途 | プライバシーポリシー |
|---|---|---|
| Supabase | データベース・認証 | https://supabase.com/privacy |
| Firebase (Google) | プッシュ通知 | https://policies.google.com/privacy |
| Vercel | ホスティング | https://vercel.com/legal/privacy-policy |
| Google Fonts | フォント配信 | https://policies.google.com/privacy |

# 付録

## A. よくある質問（FAQ）

**Q. スタンプラリーが使えない**  
A. ホーム画面への追加（PWA インストール）が必要です。Safari（iOS）または Chrome（Android）からアクセスして「ホーム画面に追加」を行ってください。

**Q. 通知が届かない（iOS）**  
A. iOS では PWA としてインストールした場合のみプッシュ通知が届きます。Safari からホーム画面に追加して、アプリとして起動してください。

**Q. QR コードが読み取れない**  
A. スタンプ用 QR コードは1分ごとに更新されます。QR の写真を撮影しても2分後には無効になります。必ず会場で直接スキャンしてください。

**Q. パスワードを忘れた**  
A. ログインページ → 「パスワードをお忘れの方」からリセットメールを送信できます。

**Q. シフトが割り当てられなかった**  
A. ✕（入れない）を多く設定している場合、適切なコマへの割り当てが困難になる場合があります。担当の先生に相談してください。
