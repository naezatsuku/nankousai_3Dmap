## 2. 動作確認（デモの一本通し）

「実際に動くの？」と聞かれたとき即座に見せられる状態にしておく。

### アカウント作成〜承認フロー
- [ ] 新規アカウントをサインアップできる
- [ ] プロフィールページで「学校区分：先生」を選択・保存できる
- [ ] 管理画面「先生管理」ページに承認待ちとして表示される
- [ ] 「先生として承認」ボタンで teacher ロールに昇格できる
- [ ] 承認後、先生管理ページの一覧に表示される
- [ ] 担当クラスを割り当てられる

### 先生として管理画面を使う
- [ ] ログイン後、TEACHER バッジが表示される
- [ ] ダッシュボードにアクセスできる
- [ ] 展示編集（担当クラスのみ表示される）
- [ ] お知らせ一覧・作成・編集（担当クラスのみ）
- [ ] 担当外のクラスの編集ページに直接URLでアクセスしようとしてもブロックされる
- [ ] ユーザー管理・先生管理ページには入れない
- [ ] シフト管理ページには入れない

### 変更ログ
- [ ] 展示を編集すると変更ログに記録される
- [ ] お知らせを投稿すると記録される
- [ ] クイック更新（待ち時間）すると記録される
- [ ] 先生ログインで「変更ログ」ページが表示される
- [ ] クラスごと・操作種類ごとにフィルターが機能する

### 通知設定
- [ ] 「通知設定」ページが先生ログインで表示される
- [ ] トグルのON/OFFを切り替えて保存できる
- [ ] 来場者サイトの「通知」ページを開くと先生向けセクションが表示される

### プッシュ通知
- [ ] 通知設定ページで「有効にする」をタップするとブラウザの許可ダイアログが出る
- [ ] 許可後、展示を編集すると実機に通知が届く（PCとスマホ両方で確認）
- [ ] 通知設定がOFFの操作（例：待ち時間更新）では通知が届かない

## 4. 先生に決めてもらう事項

### アカウント運用
- 先生用アカウントの発行は誰が管理するか（生徒の実行委員？ 先生自身？）
- 担当クラスは先生1人につき何クラスになる見込みか（複数担任はあるか）
- 文化祭終了後にアカウントをどうするか（削除・保持・次年度引き継ぎ）
- インフォメーションなどのその他のメニューのページをどうするか。

### 編集権限の範囲
- 展示内容の編集権限を先生に与えることへの懸念はないか
- 先生が編集できてほしくない項目はあるか（例：サムネイル画像・公開状態）
- 生徒が行った編集を先生が上書きした場合の対応方針

### 通知
- どの操作で通知を受け取りたいか（現在のデフォルト設定でよいか）
- 深夜・早朝の変更でも通知が届くことへの懸念はないか（時間帯制限が必要か）
- 通知音・バイブレーションの設定は端末任せでよいか
- スマートフォン・PCどちらで受け取ることが多いか

### 変更ログ
- 変更ログはどの程度の期間保持すればよいか
- 変更者が「不明なユーザー」と表示される場合への対応は必要か
- 先生が自分で行った編集もログに残ることへの確認

### セキュリティ・プライバシー
- 先生アカウントに学校のメールアドレスを使う想定か
- 先生の名前（フルネーム）をシステムに登録することへの懸念はないか

## 4. 企画書の未記載事項

先生から質問が来たときに答えられるよう把握しておく。

| 質問されそうなこと | 答え |
|---|---|
| シフトには関わるか | 関わらない設計（先生ロールはシフト機能にアクセスできない） |
| 他クラスの状況も見えるか | 見えない。自分の担当クラスのみ |
| 通知はいつ届くか | 変更保存と同時（遅延なし） |
| 管理画面に入らないと通知設定できないか | 来場者サイトの「通知」ページからも設定可能 |
| 承認を取り消せるか | 管理者が「→ TEACHER に戻す」または削除で対応可能 |
| ログインURLはどこか | 来場者サイトと同じドメインの /admin/login |
| パスワードを忘れたらどうするか | ログインページの「パスワードを忘れた方」から再設定メールを送れる |
| iPadやPCでも使えるか | ブラウザがあれば端末を問わず使える |
| 複数のデバイスで同時にログインできるか | できる |
| 生徒が先生の変更ログを見られるか | 見られない（先生ロール専用ページ） |

## 5. 想定されるフィードバックと対応案

| 先生から来そうな意見 | 対応案 |
|---|---|
| 「自分で承認するのが面倒」 | 招待メール方式（管理者がメールを送り先生が直接登録）への切り替えも可能 |
| 「通知が多すぎると困る」 | デフォルトを全OFFにすることも可能 |
| 「通知が来なさすぎると困る」 | デフォルトを全ONにすることも可能 |
| 「通知設定が複雑」 | 設定ページを廃止し、全通知を固定ON/OFFにするシンプル案も提示できる |
| 「生徒に勝手に編集されたくない」 | 変更ログで誰がいつ何を編集したか追跡できることを説明 |
| 「スマホに通知が来るか不安」 | デモ実機で実際に通知を届けて見せる |
| 「先生が編集できる範囲が広すぎる」 | 閲覧のみのロールを別途作ることも可能 |
| 「文化祭当日に使いこなせるか不安」 | 当日前にテスト運用期間を設けることを提案する |
| 「他の先生への説明が大変」 | 操作マニュアル（PDF）を別途作成することを提案する |
| 「学校のメールが使えない」 | Gmail 等の個人メールでも登録可能（制限なし） |
